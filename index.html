EMPEZAR

    API Endpoint

        http://10.0.0.196/

Esta API es un servicio web que permite a los desarrolladores obtener información sobre los empleados de la empresa. Asi como tambien agregar deducciones por la compra de comida.

TRAER EMPLEADOS


# Aqui un ejemplo de como traer empleados por medio de curl
curl \
-X GET http://10.0.0.196/employees \
-F 'offset=0' \
-F 'limit=5'

Para traer empleados, se debe hacer una solicitud GET al siguiente URL :
http://10.0.0.196/employees

ATRIBUTOS DEL RESULTADO

Atributo Tipo Descripción
id Integer El id del empleado. Este es unico para cada empleado.
first_name String El primer nombre del empleado.
last_name String Los apellidos del empleado.
department String El departamento al que pertenece este empleado.
turn String El turno de trabajo del empleado.

PARAMETROS DEL QUERY

Atributo Tipo Descripción
id Integer (opcional) El codigo de un empleado si desea buscar uno en especifico.
offset Integer (opcional - defecto: 0) Un cursor para usar en la paginación. La paginación comienza en 0, y cuenta cada objeto ordenado por el codigo de empleado. Ejemplo: si hay 10 empleados, y quieres los empleados 6-10, el offset es 5 y el limit es 5.
limit Integer (opcional - defecto: 100) Un limite en el numero de objetos a ser devueltos.

DEDUCCIONES


# Aqui un ejemplo de como agregar una deduccion por medio de curl
curl \
-X POST http://10.0.0.196/deductible \
-F 'employee_id=1001' \
-F 'amount=150'

Para agregar deducciones a empleados, se debe hacer una solicitud POST al siguiente URL :
http://10.0.0.196/deductible 



Resultado de ejemplo :

{
	"status": 200
}

ATRIBUTOS DEL RESULTADO

Atributo Tipo Descripción
status Integer El status de la deduccion, si es 200 se hizo correctamente, de lo contrario hubo un error refierase a la sección de ERRORES para más información.

PARAMETROS DEL QUERY

Atributo Tipo Descripción
employee_id Integer El codigo de empleado al que se le va a agregar la deduccion.
amount Float El monto de la deduccion, este debe ser un numero positivo.



Errores



            Resultado de ejemplo :
            
            {
              "error": "amount is required"
            }

En caso de cualquier error el api devuelve lo siguiente:

Atributo Tipo Descripción
error String Un mensaje con la descripción del error que sucedió
